## 安装与系统要求

| 项 | 要求 |
|----|------|
| 系统 | Windows 10/11 x64 |
| 微信 | 读聊天：需 **知聊** 开启 HTTP API；发朋友圈 / 自动回复：需 PC 微信 4.1+ 已登录 |
| Python | 微信自动化首次使用会自动安装依赖；亦可手动 `pip install -r requirements-pywechat.txt` |
| 磁盘 | 建议预留 1GB+（含本地知识库与 SQLite） |

**安装步骤**

1. 下载 `PrivateKings-2.0preview-win-x64.exe`（见 [releases](../releases/) 或 GitHub Releases）。
2. 运行安装向导，完成后从开始菜单打开 **知销**。
3. 建议先安装并配置好 **知聊**（见 [知聊使用指南](知聊使用指南.md)）。

**数据目录**：`文档\PrivateKings\`（设置、知识库 Markdown、记忆、定时朋友圈任务等）

## 首次启动与授权

1. 首次打开进入 **授权** 页，输入激活码并激活。
2. 未授权时仅可进入 **设置 → 账号 → 授权管理** 修改激活码。
3. 授权通过后进入主界面；侧栏底部显示版本 **v2.0preview**。

## 连接微信数据（知聊 API）

知销 **不直接读微信数据库**，个人聊天通过 **知聊本地 HTTP API** 获取。

**知聊侧（先做）**

1. 启动知聊并完成微信连库。  
2. **设置 → Agent 连接**：启用 API，默认 `http://127.0.0.1:5031`，设置 Token。

**知销侧**

1. **设置 → 数据源 → 微信数据**（或「微信」子页）。  
2. 填写 **API 地址** 与 **Token**（与知聊一致）。  
3. 点击 **测试连接**，显示成功即可。  
4. **会话** 页应能同步个人聊天列表；**工作台** 可选会话生成回复建议。

> 两应用为 **独立进程**：须同时运行知聊（提供 API）与知销（UI 与 AI）。

联调步骤详见 [联调说明](联调说明.md)。

## 界面与功能

**侧栏分组**

| 分组 | 入口 | 说明 |
|------|------|------|
| 工作 | **工作台** | 选会话、看消息、AI 回复建议（自动 / 快速 / 精准） |
| 工作 | **会话** | 联系人列表、WeFlow 同步 / JSON 导入 |
| 自动化 | **微信自动化** | 回复自动化、发朋友圈、定时任务 |
| 智能 | **角色** | 多销售人设与策略 |
| 智能 | **知识库** | Wiki / SOP、联系图谱、整理中心 |
| 智能 | **记忆** | 客户画像与每日摘要 |
| 系统 | **接入** | 各平台数据源状态 |
| 系统 | **设置** | AI、授权、Agent API、实时建议等 |

### 工作台 · 回复建议

1. 在 **会话** 或工作台选择 **个人聊天**（V1 不支持群聊自动建议）。  
2. 选择 **回复模式**：  
   - **自动**：推荐；事实走知识库，流程走 SOP  
   - **快速**：首响草稿，轻量检索  
   - **精准**：完整 RAG + 证据门控  
3. 点击生成建议，可复制主推回复；支持 **悬浮窗** 精简展示。

### 会话 · JSON 导入

除 API 外，可将导出的 JSON 消息 **导入** 到知销，与 API 结果合并使用。

## 微信自动化

路径：**自动化 → 微信自动化**

### 回复自动化

1. 打开 **回复自动化**（侧栏二级第一项）。  
2. 启用总开关，将联系人加入 **白名单**（填写 PC 微信可搜索到的显示名）。  
3. 新消息进入队列 → AI 生成完整建议 → 校验后 **单线程** 发送。  
4. 默认 **关闭** 全自动发送；仅对白名单个人生效。

### 朋友圈

1. 打开 **朋友圈**（侧栏二级第二项）。  
2. **发朋友圈 Beta**：编辑文案 / 选图、AI 生成、立即发布。  
3. **定时任务**：管理定时发布计划。  
4. 若检测未通过，先点 **启动/修复微信自动化**；发布期间勿遮挡微信窗口。

**环境**：Windows + 本机微信 + Python 3.10+（自动化组件）。

## 知识库与记忆

### 知识库

- 知识以 **Markdown** 文件存放在 `Documents/PrivateKings/knowledge/`。  
- 类型：**Wiki**（事实）与 **SOP**（流程脚本）。  
- 页内可 **重建索引**、整理中心审核提案、**AI 读取体检** 排查检索质量。

### 记忆

- 按联系人维护画像、事实、异议、承诺、每日摘要。  
- 可在 **记忆** 页手动编辑；生成建议后可按设置自动更新。

## Agent API（可选）

知销自带 HTTP API（默认 `http://127.0.0.1:61743`），供 Cursor Agent、脚本调用。

- **设置 → Agent Skill 与 API** 可查看端口、Token、下载内置 Skill。  
- 健康检查：`GET /health`（无需鉴权）。

## 常见问题

| 现象 | 处理 |
|------|------|
| 会话列表为空 | 检查知聊 API 是否开启、Token 是否正确、知聊是否在运行 |
| 授权失效 | 设置 → 授权管理 → 重新激活或联系分发方 |
| 自动化发不出 | 确认微信已登录、Python 环境、先点「启动/修复微信自动化」 |
| 页面无法滚动 | 2.0preview 已修复布局滚动；升级后仍异常可反馈 Issue |
| 群聊无建议 | V1 仅支持 **个人聊天**，群聊为预留能力 |
