# 知聊 + 知销 联调说明

两应用 **完全独立进程**，通过本机 HTTP 协作：知聊读微信 WCDB，知销读知聊 API 并做 AI 回复。

## 端口一览

| 服务 | 默认地址 | 用途 |
|------|----------|------|
| 知聊 HTTP API | `http://127.0.0.1:5031` | 会话 / 消息 / SSE 推送 |
| 知销 Agent API | `http://127.0.0.1:61743` | 外部 Agent、脚本（可选） |

## 推荐步骤

| 步骤 | 操作 |
|------|------|
| 1 | 源码仓库或安装包启动 **知聊** |
| 2 | 知聊完成微信连库 |
| 3 | 知聊 **设置 → Agent 连接**：启用 API，设置 Token |
| 4 | 验证：`GET http://127.0.0.1:5031/api/v1/status`（带 Bearer Token） |
| 5 | 启动 **知销** |
| 6 | 知销 **设置 → 微信数据**：填 API 地址 + Token → 测试连接 |
| 7 | 知销 **会话** 页刷新，应出现个人聊天列表 |
| 8 | **工作台** 选会话，测试回复建议 |

## 开发模式（源码）

在 monorepo 根目录分别开两个终端：

```bash
npm run dev                 # 知聊 Vite 3000 + Electron
npm run private-kings:dev   # 知销 Vite 3717 + Electron
```

## 数据目录（互不覆盖）

| 应用 | 路径 |
|------|------|
| 知聊 | `%USERPROFILE%\Documents\WeFlow\` |
| 知销 | `%USERPROFILE%\Documents\PrivateKings\` |

## 注意

- 知销 **不能** 在知聊未运行时通过 API 读微信库（除非使用 JSON 导入）。  
- 微信自动化（发朋友圈 / 自动回复）走知销主进程 Python，**不依赖** 知聊进程，但仍需本机微信。  
- 授权 **独立**：知聊（zhiliao）与知销（zhixiao）各有一套激活数据。
